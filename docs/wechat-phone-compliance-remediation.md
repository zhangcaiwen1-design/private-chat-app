# 微信小程序手机号合规整改说明

## 整改原因

微信平台通知指出小程序存在违规收集用户手机号的问题。当前整改目标是取消首次进入时的手机号必填要求，避免将手机号作为使用小程序的前置条件。

## 已完成调整

- 小程序首次进入改为微信登录，通过 `wx.login` 获取登录凭证后创建或登录账号。
- 首次进入页面不再要求填写手机号，也不再调用手机号授权能力。
- 本地计算器进入密码仅保存在用户设备本地，不作为手机号收集理由。
- 手机号调整为设置页的选填绑定项，仅用于用户主动选择通过手机号添加好友、账号识别或客服核验。
- 后端新增 `/api/v1/auth/wechat-login`，支持微信登录账号。
- 后端保留旧手机号登录接口，用于兼容历史用户，不作为小程序默认入口。

## 微信后台和服务器配置

1. 微信公众平台 -> 开发管理 -> 开发设置 -> 开发者ID：
   - 确认 AppID 为 `wx54ba54fb4d2b3aba`。
   - 获取并妥善保存 AppSecret。
2. 线上后端环境变量：
   - `WECHAT_MINIPROGRAM_APP_ID=wx54ba54fb4d2b3aba`
   - `WECHAT_MINIPROGRAM_APP_SECRET=微信后台的 AppSecret`
3. 微信公众平台 -> 开发管理 -> 开发设置 -> 服务器域名：
   - request 合法域名需包含 `https://privatechat.yifan1.com`。
4. 上传前本地检查：
   - `npm run wechat:check`
   - 服务器配置完成后执行 `npm run wechat:check:prod`

## 给微信后台的整改反馈文案

1. 小程序登录入口已由手机号登录/注册改为微信登录。

3. 手机号仅保留为用户在设置页主动选择的选填绑定信息，用于添加好友、账号识别或客服核验。
