# Apple 审核测试流程

更新时间：2026-06-13

## 测试账号

- 手机号：<待填写审核测试手机号>
- 本机进入密码：<待填写测试密码>
- 会员状态：iOS 首版不要求会员即可体验聊天
- 测试联系人：建议提前准备 1 个联系人，避免审核员卡在空列表

## 审核员可操作路径

1. 打开 App。
2. 如出现登录页，输入测试手机号和本机进入密码。
3. 进入聊天列表。
4. 打开已有联系人，发送文字消息。
5. 点击输入栏旁的更多按钮，选择图片或拍照发送。
6. 切换语音模式，录制并发送语音消息。
7. 返回聊天列表，打开二维码/添加好友入口。
8. 打开云端记录，查看可保存或恢复的聊天数据。
9. 点击锁定，确认 App 回到锁定/登录状态。

## 截图清单

至少准备 iPhone 6.7 英寸截图：

1. 登录页。
2. 聊天列表。
3. 聊天窗口文字消息。
4. 聊天窗口图片/语音能力。
5. 添加好友或二维码页。
6. 云端记录页。
7. 设置或本机锁定页。

## 截图文案限制

不要在截图或描述里出现：

- 伪装
- 隐藏
- 防查
- 躲避
- 偷偷
- 秘密关系
- 绕过审核

推荐使用：

- 私密沟通
- 本机锁定
- 云端记录
- 双人聊天
- 联系人管理
- 账号同步

## 提交前自测

```powershell
npm run lint
npm run site:build
```

正式协议站点构建必须通过：

```powershell
$env:PUBLIC_LEGAL_ENTITY="<主体名称>"
$env:PUBLIC_SUPPORT_EMAIL="<客服邮箱>"
npm run site:build:prod
```

如果没有填写主体名称和客服邮箱，`site:build:prod` 会失败，这是预期行为。
