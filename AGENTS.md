## Collaboration rules

- 本项目默认自主开发。你可直接读代码、改代码、跑测试、跑本地脚本、启动或重启本地服务，不用先打招呼，不用先汇报计划，也不用先征求许可。
- Keep user-facing progress updates minimal and concise. Prefer doing the work over narrating it.
- Do not greet, restate the request, or pause for approval before normal local development work.
- 默认直接执行；除非遇到真实阻塞，否则不要提问。需要沟通时，用最短一句话说明。
- 任务完成时，只需用一两句话说明结果和下一步，不要写长篇总结。
- 只有以下情况才需要确认：删除文件、批量覆盖文件、可能覆盖未知用户工作、改 git 历史、push/PR、改第三方服务、生产或线上数据操作。
- Ask for confirmation before any action that affects shared or external state, including `git push`, creating pull requests, posting comments, sending messages, deploying, changing CI/CD, or modifying third-party services.
- Prefer Chinese when communicating in this project.
